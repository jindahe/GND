## P8：quimb 后端实验（已完成）

新增两个文件：

- **`src/core/quimb_contraction.py`**：基于 quimb 的 bMPS 后端
  - `_compress_mps_quimb`：先 right-canonize（QR，从右到左），再 SVD 压缩（从左到右），即双向压缩
  - `quimb_contract_region` / `quimb_prob_with_parity`：与 `bmps_contraction.py` 同接口
  - `calculate_CMI_quimb`：MC 估计器入口

- **`src/scripts/benchmark_quimb_backend.py`**：三类基准测试（region 精度、CMI 回归、大 r 性能）

### 关键实验数据

| 测试项 | 结论 |
|--------|------|
| Region-level 精度（12项对比） | quimb 优于 custom bMPS 的占 9/12 |
| 速度对比 | custom bMPS 比 quimb **快 5 倍** |
| CMI r=4（N=20，chi=32） | custom: +0.003，quimb: −0.009 |

### 结论

**结论 B：仅作为 correctness oracle 使用**

- 双向压缩（quimb）并未改善 CMI 层面的结果
- **根本原因：chi 本身不够大**，而非压缩方向问题
- quimb 不引入主生产流程，仅用于关键点的交叉验证

## chi 收敛性扫描：实验数据汇总

### 测试范围

`p ∈ {0.05, 0.11, 0.15}`，`r ∈ {6, 8, 10}`，`chi ∈ {32, 64, 96}`，每点 2000 样本

### 关键结论

| 场景 | chi=32 (p=0.15) | chi=64 (p=0.15) | chi=96 (p=0.05, r=6, 4个chunk) |
|------|-----------------|-----------------|-------------------------------|
| r=6  | −0.019 | −0.0055 | +0.003 ~ +0.009（正值） |
| r=8  | — | −0.034 | — |
| r=10 | — | −0.107 | — |

- **chi=32**：r≥5 时系统性负偏，不可用
- **chi=64**：r=6 的 p=0.11 已给出正值（+0.0073），p=0.15 仍为负
- **chi=96**：r=6 下 p=0.05/0.11 初现正值，但样本极少（4个chunk×200），尚需更多样本验证

### 截断偏差的 chi 依赖关系（r=6，p=0.15）

chi=32 → −0.019，chi=64 → −0.0055，改善比 ≈ 3.8 ≈ 2²，即偏差 ∝ chi⁻²。
外推：chi=96 偏差 ≈ −0.002，chi=128 偏差 ≈ −0.0012。

### 时间模型

以实测 chi=96，r=6（~22 s/样本）为基准：

$$t(r, \chi) = 22\ \text{s} \times \frac{C_r}{C_6} \times \left(\frac{\chi}{96}\right)^3, \quad C_r = 3(2+4r)^2 + 3(2+2r)^2$$

### 推荐 chi 配置

| r | chi | t/样本（估算） |
|---|-----|--------------|
| 1–3 | exact TN | 0.01–0.03 s |
| 4 | 64 | ~1.6 s |
| 5 | 64 | ~4.7 s |
| 6 | 96 | ~22 s |
| 7 | 128 | ~69 s |
| 8 | 128 | ~89 s |

## GPU 加速可行性评估

### 测试环境

- GPU：NVIDIA RTX 5090（32 GB，Blackwell，CUDA 12.8）
- CPU：Intel Xeon Platinum 8470Q（104 物理核）
- 瓶颈操作：对 256×128 矩阵（chi=128）做 SVD

### 实测数据

| 操作 | CPU float64 | GPU float64 | GPU float32 |
|------|------------|------------|------------|
| SVD 单次（256×128） | 58 ms | 42 ms（+1.4×） | 3.1 ms（+18×） |
| SVD 批量 batch=32 | — | **42 ms/SVD（无加速）** | 5.3 ms/SVD |
| SVD 批量 batch=96 | — | **42 ms/SVD（无加速）** | 5.3 ms/SVD |
| SVD 批量 batch=256 | — | 42 ms/SVD | — |
| einsum（chi=128） | 2.09 ms | 0.33 ms（+6×） | ~0.1 ms |

### 三个关键发现

1. **RTX 5090 的 float64 SVD 几乎无加速（1.4×）**
   消费级 GPU 的 FP64 算力被硬件限制为 FP32 的 1/64，对 (256×128) 矩阵仅快 1.4 倍

2. **GPU 批量 SVD 完全失效**
   batch=32/96/256 的每次 SVD 耗时均为 42 ms，与单次相同。
   cuSOLVER 对该尺寸矩阵的批量 SVD 内部串行执行，批量化无意义。

3. **96 核 CPU 并行处理量远超 GPU**

   | 方案 | 吞吐量 |
   |------|--------|
   | CPU 96 进程并行 | **1.08 样本/秒** |
   | GPU float64（替代单个 worker） | 0.016 样本/秒（慢 **68 倍**） |

### 结论

**当前 P9 任务不引入 GPU，继续使用 96 CPU 进程并行。**

GPU 在以下条件下才值得重新评估：
- chi ≥ 512（SVD 矩阵达到 1024×512，GPU 利用率才够高）
- 使用数据中心 GPU（A100/H100，FP64 性能为 FP32 的 1/2）
- 全面切换 float32 精度并验证临界点准确性
