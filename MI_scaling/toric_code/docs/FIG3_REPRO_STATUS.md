## 下一步运算计划

## Phase 1 — 固化 Fig.3(d) 主版本

目标：得到一版可直接放进草稿的 `CMI`–`r` 图与主表。

### 1.1 生成最终主图输入

- 使用 `outputs/CMI_paper_repro_main_table_v1.csv`
- 使用 `outputs/CMI_p10_refit_with_systematics_v1.csv`
- 统一主图口径：
  - `p=0.05`：使用 `r=1..6`
  - `p=0.11`：使用 `r=1..6` 与 `r=2..6` 并列
  - `p=0.15`：主口径优先 `r=1..6 weighted_mc`，辅助口径给出 `r=1..5 weighted_eff`

### 1.2 产出图级说明

在图注或说明文档中明确写出：

- `p=0.15, r=4` 已由 `chi=64` 修正到 `chi=160`；
- `p=0.15, r=5` 采用 `chi=192`；
- `r=7` 还未达到纳入最终拟合集的条件。

### 1.3 不再继续推进的方向

短期内不建议继续：

- 把 `p=0.15, r=5` 再升到 `chi=256`；
- 直接把 `r=7` 补到 `N=5000`；
- 在未统一系统误差口径前重新做 collapse。

## Phase 2 — 推进 Fig.3(b) 的 `p` 扫描复现

目标：得到一版以 `r` 为参数、`CMI` 随 `p` 变化并在 `p_c \approx 0.11` 附近出现峰值的图。

### 2.1 推荐的 `r` 集合

优先做：

- `r = 1, 2, 3, 4, 5, 6`

暂不纳入：

- `r = 7, 8`

### 2.2 推荐的 `p` 网格

分两层：

- 粗网格：`p = 0.03, 0.05, 0.07, 0.09, 0.11, 0.13, 0.15, 0.17`
- 临界区细网格：`p = 0.095, 0.10, 0.105, 0.11, 0.115, 0.12, 0.125`

建议先做粗网格，再只对峰区补细网格。

### 2.3 每点样本数路线

稳步推进建议：

1. **Scout 阶段**：每点 `N=2000`
2. **主生产阶段**：峰区与边界区补到每点 `N=10000`
3. **论文图阶段**：对最终保留点补到每点 `N>=3.5e4`

这样做的好处：

- 先定位峰位和可疑点；
- 再把算力集中投入对最终图真正重要的点；
- 避免把大量样本浪费在明显平滑区。

### 2.4 `chi` 口径

为避免再次出现 `p=0.15, r=4` 这类系统误差，建议：

- `r<=3`：维持当前低 `chi` / exact TN 口径；
- `r=4`：至少用 `chi=160`；
- `r=5`：至少用 `chi=192`；
- `r=6`：至少用 `chi=256`；

也就是说，后续完整 `p` 扫描不应继续混用旧的 `r=4,5,6` 低 `chi` 数据。

## Phase 3 — 推进 Fig.3(c) 的 collapse 复现

目标：在统一系统误差口径下，尝试恢复

- `p_c \approx 0.11`
- `\nu \approx 1.8`
- `\alpha \approx 1.1`

### 3.1 进入 collapse 的前提

只有满足下面三条后才建议进入：

- `r=1..6` 的 `p` 扫描已完成；
- `r=4,5,6` 的高 `chi` 口径已统一；
- 峰区 `p \in [0.095, 0.125]` 的统计误差已降到能区分峰位漂移。

### 3.2 collapse 的稳步推进方式

建议分三步：

1. 固定 `p_c = 0.11`，仅拟合 `\alpha`
2. 再允许 `p_c, \alpha` 同时浮动
3. 最后再放开 `\nu`

不要一开始就三参数全放开，否则很容易被有限尺寸和系统误差带偏。

### 3.3 窗口选择

建议并列测试：

- `r = 1..5`
- `r = 1..6`

不要把 `r=7` 强行混入 collapse，除非其 scout 被重新证实为稳定。

---

## Phase 4 — 论文图复现的最终交付顺序

建议按下列顺序推进，而不是并行乱算：

1. **先固化 Fig.3(d)**  
   目标：一版可 defend 的 `CMI`–`r` 主图与表格。
2. **再补 Fig.3(b)**  
   目标：峰值位置、峰高随 `r` 变化的稳定图。
3. **最后做 Fig.3(c)**  
   目标：collapse 与指数参数估计。

这样推进的原因：

- Fig.3(d) 直接暴露 truncation bias，是当前最关键的数值可信度基础；
- Fig.3(b) 需要更多点，但不必先知道 collapse 参数；
- Fig.3(c) 对前两者质量最敏感，适合最后做。
